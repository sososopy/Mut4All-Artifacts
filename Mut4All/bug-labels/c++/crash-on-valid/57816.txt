Issue Title:
[clang] Inline [[clang::always_inline]] crashes when applied to a function with the same name (but a different overload)

Source Code & Bug Trace:
void f(int*, int) {}

template<class T>
void f(T t) {
  [[clang::always_inline]] f(t, 0);
}

int main() {
  f(nullptr);
}

Bug Labels:
clang:frontend, crash-on-valid
