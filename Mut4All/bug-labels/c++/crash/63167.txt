Issue Title:
Clang frontend C++ crash when using automatic variables directly in  noexcept  of generic lambda

Source Code & Bug Trace:
// clang++ -std=c++20 -fsyntax-only
struct true_type { constexpr operator bool() const { return true; } };

constexpr auto f() {
    true_type i;
    return [=](auto) noexcept(i) -> bool {
        return true;
    };
}

// static_assert(f()(0));
